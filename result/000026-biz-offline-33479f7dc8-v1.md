## 1. 核心内容

**请求报文结构**

{
  "request": {
    "head": {
      "sysId": "uops",
      "msgId": "uops031.000.01",
      "sendDate": "YYYY-MM-DD HH:mm:ss",
      "senderInstId": "发起方机构标识",
      "receiverInstId": "接收方机构标识",
      "direction": "01"
    },
    "body": {
      "startTime": "查询开始时间",
      "endTime": "查询结束时间",
      "infoNo": "运维信息编号"
    },
    "signature": "签名内容"
  }
}

**应答报文结构**

{
  "response": {
    "head": {
      "sysId": "uops",
      "msgId": "uops031.000.01",
      "sendDate": "YYYY-MM-DD HH:mm:ss",
      "senderInstId": "发起方机构标识",
      "receiverInstId": "接收方机构标识",
      "direction": "01",
      "rspCd": "0000",
      "rspMsg": "应答消息"
    },
    "body": {
      "infoNo": "信息编号",
      "instName": "发起方单位名称",
      "instId": "发起方金融编码",
      "reinstName": "接收方单位名称",
      "reinstId": "接收方金融编码",
      "startTime": "开始时间",
      "endTime": "结束时间",
      "reason": "原因",
      "contactPerson": "联系人",
      "contactPhone": "联系电话"
    },
    "signature": "签名内容"
  }
}

## 2. 适用边界

本报文用于根据运维信息编号进行查询，需提供查询的时间范围和具体的运维信息编号。

## 3. 使用要求

1. 请求报文必须包含完整的签名内容。
2. 应答报文中的 `rspCd` 为 "0000" 表示成功，其他值表示失败，具体含义见 `rspMsg`。
3. 查询开始时间 (`startTime`) 和查询结束时间 (`endTime`) 必须在请求报文的 `body` 中指定。
