## 1. 核心内容

**请求报文结构**

{
  "request": {
    "head": {
      "sysId": "uops",
      "msgId": "uops011.000.01",
      "sendDate": "YYYY-MM-DD HH:mm:ss",
      "senderInstId": "发起方机构标识",
      "receiverInstId": "接收方机构标识",
      "direction": "01"
    },
    "body": {
      "mTrxCtgyCd": "01",
      "mAccTpCd": "01",
      "succCnt": "交易成功数量",
      "succAmt": "交易成功金额",
      "failCnt": "交易失败数量",
      "failAmt": "交易失败金额",
      "rate": "成功率"
    },
    "signature": "签名内容"
  }
}

**应答报文结构**

{
  "response": {
    "head": {
      "sysId": "uops",
      "msgId": "uops011.000.01",
      "sendDate": "YYYY-MM-DD HH:mm:ss",
      "senderInstId": "发起方机构标识",
      "receiverInstId": "接收方机构标识",
      "direction": "01",
      "rspCd": "0000",
      "rspMsg": "应答消息"
    },
    "body": {},
    "signature": "签名内容"
  }
}

## 2. 适用边界

适用于成员单位和网联平台之间通过交易监控统计信息报文接口方式，传递交易监控统计信息。报文结构包含请求报文和应答报文两种类型。

## 3. 使用要求

**请求报文使用要求**：
1. 请求报文必须包含 head 和 body 结构。
2. head 中必须包含 sysId、msgId、sendDate、senderInstId、receiverInstId、direction 字段。
3. body 中必须包含 mTrxCtgyCd、mAccTpCd、succCnt、succAmt、failCnt、failAmt、rate 字段。
4. 必须包含 signature 字段。

**应答报文使用要求**：
1. 应答报文必须包含 head 和 body 结构。
2. head 中必须包含 sysId、msgId、sendDate、senderInstId、receiverInstId、direction、rspCd、rspMsg 字段。
3. body 必须为空对象 {}。
4. 必须包含 signature 字段。
