## 1. 核心内容

**请求报文结构**

{
  "request": {
    "head": {
      "sysId": "uops",
      "msgId": "uops051.000.01",
      "sendDate": "YYYY-MM-DD HH:mm:ss",
      "senderInstId": "发起方机构标识",
      "receiverInstId": "接收方机构标识",
      "direction": "02"
    },
    "body": {
      "sceneId": "03",
      "content": {
        "instName": "账户侧单位名称",
        "instId": "账户侧金融编码",
        "succRate": "成功率",
        "exceptionDesc": "异常描述",
        "returnCode": "异常返回码",
        "serialNo": "流水号"
      }
    },
    "signature": "签名内容"
  }
}

**应答报文结构**

{
  "response": {
    "head": {
      "sysId": "uops",
      "msgId": "uops051.000.01",
      "sendDate": "YYYY-MM-DD HH:mm:ss",
      "senderInstId": "发起方机构标识",
      "receiverInstId": "接收方机构标识",
      "direction": "02",
      "rspCd": "0000",
      "rspMsg": "应答消息"
    },
    "body": {},
    "signature": "签名内容"
  }
}

## 2. 适用边界

本场景适用于自由格式报文的场景三，用于传输账户侧单位名称、金融编码、成功率、异常描述、异常返回码及流水号等信息。应答报文用于返回处理结果。

## 3. 使用要求

- 请求报文中 `direction` 字段值为 "02"。
- 请求报文中 `sceneId` 字段值为 "03"。
- 请求报文和应答报文均包含 `signature` 字段，值为 "签名内容"。
- 应答报文 `body` 字段为空对象 `{}`。
