## 1. 核心内容

> - "signature": "签名内容"
B.12 自由格式报文-场景八定义了该特定场景下的请求与应答报文结构，报文整体采用 JSON 格式，包含 request（请求）或 response（应答）下的 head、body 和 signature 三个主要部分。

**请求报文结构：**
- **head（报文头）**：
  - `sysId`：固定为 "uops"
  - `msgId`：固定为 "uops051.000.01"
  - `sendDate`：发送日期时间，格式为 "YYYY-MM-DD HH:mm:ss"
  - `senderInstId`：发起方机构标识
  - `receiverInstId`：接收方机构标识
  - `direction`：固定为 "01"
- **body（报文体）**：
  - `sceneId`：固定为 "08"
  - `content`：包含以下具体业务字段：
    - `instName`：发起方单位名称
    - `instId`：发起方金融编码
    - `reinstName`：接收方单位名称
    - `reinstId`：接收方金融编码
    - `infoNo`：信息编号
    - `startTime`：开始时间
- **signature（签名）**：签名内容

**应答报文结构：**
- **head（报文头）**：
  - `sysId`：固定为 "uops"
  - `msgId`：固定为 "uops051.000.01"
  - `sendDate`：发送日期时间，格式为 "YYYY-MM-DD HH:mm:ss"
  - `senderInstId`：发起方机构标识
  - `receiverInstId`：接收方机构标识
  - `direction`：固定为 "01"
  - `rspCd`：应答代码，示例值为 "0000"
  - `rspMsg`：应答消息
- **body（报文体）**：内容为空对象 `{}`
- **signature（签名）**：签名内容

## 2. 适用边界

本报文结构规范仅适用于 B.12 自由格式报文-场景八（sceneId 为 "08"）。关于该场景所对应的具体业务含义和触发条件，原文未明确说明。

## 3. 使用要求

- 发起请求时，必须严格按照规定的 JSON 结构组装数据。
- `sendDate` 字段的取值必须严格遵循 `YYYY-MM-DD HH:mm:ss` 的时间格式。
- 请求报文的 `sceneId` 必须赋值为 "08"。
- 应答报文的 `body` 节点默认返回空对象 `{}`，具体的业务响应状态通过 `head` 中的 `rspCd` 和 `rspMsg` 传递。
