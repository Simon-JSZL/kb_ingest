## 1. 核心内容

本规范在报文结构的“属性”中，使用 `[m..n]` 的形式描述业务要素的强制项、可选项和多重性，表示该要素最少出现 m 次，最多出现 n 次。其中，强制项 m 为 1，可选项 m 为 0，n 为大于 0 的整数。对于可选项的字段，报文传递时，若 value 没有值，则该字段的 key 不参与报文组装。

## 2. 适用边界

该规则适用于网络支付清算平台联合运维互联互通技术规范下的报文结构定义，具体作用于报文结构中的“属性”字段，用于规范业务要素在报文中的存在状态和出现次数。原文未明确说明其他不适用范围。

## 3. 使用要求

在进行报文组装和传递时，必须遵循以下要求：
1. 必须依据 `[m..n]` 的形式判断对应业务要素是否必须存在以及可出现的次数上限。
2. 当处理可选项字段（即 m 为 0 的字段）时，必须先判断其 value 是否有值。
3. 若判断结果为 value 没有值，禁止将该字段的 key 参与报文组装。
